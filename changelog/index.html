Change log

Version 2.0

Change for description field

The description field got an upgrade to a full wysiwyg field with the following allowed tags: <em><strong><code><ul><ol><li><dl><dt><dd><h2><h3><h4><h5><h6><img><h1><pre><p><a><table><caption></caption><tbody><thead><tfoot><th><td><tr><br>.

In Open Data V1 there were just <p> tags allowed.

More possible properties

The following properties are now available in the API:

  • priceRange
  • starRating
  • openingHoursSpecification
  • openingHours
  • amenityFeature
  • award
  • offers
  • paymentAccepted
  • currenciesAccepted
  • checkinTime
  • checkoutTime
  • petsAllowed
  • numberOfRooms
  • maximumAttendeeCapacity

Image

Image is now delivered with type ImageObject and the properties “caption”.

Category

The category is now delivered with the disocvers.wiss categories and their IDs. /api/category.json
